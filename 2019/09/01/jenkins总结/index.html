jenkins总结

2019-9-1

Jenkins是一款开源 CI&CD 软件，用于自动化各种任务，包括构建、测试和部署软件.
Jenkins 支持各种运行方式，可通过系统包, Docker 或者通过一个独立的 Java 程序.

linux 不区分后缀，都是以文件保存的包括硬件。。区分了只是为了认为区分。。虚拟机不能超过真实机的一半。

驱动器分区：
/home 2000 （启动所需）
/boot 200 (启动所需)
/seap 1000 (系统所用)
/ 剩余所有 （以后的所有操作）

启动：net start jenkins
关闭：net stop jenkins

https://plugins.jenkins.io/ 插件手动下载地址

https://blog.csdn.net/wei_ada/article/details/106644019
mac：

安装最新的LTS版本： brew install jenkins-lts
安装特定的LTS版本： brew install jenkins-lts@YOUR_VERSION
启动jenkins服务： brew services start jenkins-lts
关闭： brew services start jenkins-lts
重新启动Jenkins服务： brew services restart jenkins-lts
更新Jenkins版本： brew upgrade jenkins-lts

1.安装一次很麻烦，各种插件各种依赖，下半天。

根据jinkensFile 里面的代码来构建：
agent: 指定节点
options：构建的一些选项
stage:构建阶段
post：构建后的操作

2.配置各种命令启动微服务也很麻烦。
需要运维来写shell命令来执行。
水流线语法+docker要好一点。
